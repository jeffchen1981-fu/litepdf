# MuPDF 1.27.2 Stage 2 — Upgrade Landing (search behavior identical) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pin MuPDF 1.24.11 → 1.27.2 and prune the exe back under the 19 MB gate, with `src/` untouched so search source and output stay identical to today (Tier-1 search activation is the separate post-v1.0 Stage 3).

**Architecture:** Two functional changes — bump the `third_party/mupdf` submodule pointer to tag `1.27.2`, and rewrite the `cmake/ImportMuPDF.cmake` prune set from V2 (5 defines) to the spike-proven V9 set (16 defines, incl. `TOFU_NOTO`) plus a new configure-time prune-effective assertion. No C++ changes: `Document::page_hits` keeps calling `fz_search_page`, the `(void)flags; (void)abort_flag;` no-ops remain, and the `[!shouldfail]` case-sensitive tripwire test stays green. The work is build-config + dependency-pin, so the "tests" are the existing CI gates (173 unit tests, size ceiling, smoke, fixture `--check`, benchmark self-test) plus a manual visual determinism review — not new unit tests (there is no new code logic to test-first).

**Tech Stack:** Git submodule, CMake `ExternalProject` driving MSBuild on MuPDF's `mupdf.sln`, MSVC v143 `/MT`, Catch2/ctest, PowerShell 5.1-safe gate scripts, reportlab fixture generators. **Shell note:** the executor runs **Windows PowerShell 5.1**; binary stdout must be captured via `cmd /c` (PS `>` re-encodes and corrupts it), and no `?.`/`??`/ternary in any `.ps1`.

## Task 2: Rewrite the prune set to the spike-proven V9 + add the prune-effective assertion

**Files:**
- Modify: `cmake/ImportMuPDF.cmake:79-118` (the `# --- Phase 11.5: prune …` block)

- [ ] **Step 1: Replace the V2 define set and marker with V9**

In `cmake/ImportMuPDF.cmake`, replace the `_PRUNE_VER` + `_prune` definitions (currently lines ~91-98) with the V9 set transplanted verbatim from `spike/mupdf-1.27`:

```cmake
set(_MUPDF_CONFIG_H "${_MUPDF_ROOT}/include/mupdf/fitz/config.h")
# V9 = the spike-proven durable set (spike findings lever table, 2026-06-09).
# V3-V8 were throwaway experiment rungs on spike/mupdf-1.27 and never landed;
# this is the first durable bump after V2, jumping straight to V9 so the marker
# matches the findings doc the team will reference.
set(_PRUNE_VER "LITEPDF_PRUNE_V9")   # bump when the injected define set changes
set(_prune
    "/* ${_PRUNE_VER} — injected by ImportMuPDF.cmake (do not edit) */\n"
    "#define FZ_ENABLE_JS 0\n"
    "#define FZ_ENABLE_OCR_OUTPUT 0\n"
    "#define FZ_ENABLE_DOCX_OUTPUT 0\n"
    "#define FZ_ENABLE_BARCODE 0\n"
    "#define FZ_ENABLE_BROTLI 0\n"
    "#define FZ_ENABLE_HYPHEN 0\n"
    "#define FZ_ENABLE_HYPHEN_ALL 0\n"
    "#define FZ_ENABLE_OFFICE 0\n"
    "#define FZ_ENABLE_MOBI 0\n"
    "#define FZ_ENABLE_HTML 0\n"
    "#define FZ_ENABLE_TXT 0\n"
    "#define FZ_ENABLE_IMG 0\n"
    "#define TOFU_SYMBOL\n"
    "#define TOFU_EMOJI\n"
    "#define TOFU_NOTO\n"
    "#define TOFU_CJK_LANG\n")
```

> **Why each lever (spike findings M2 + format-support investigation):** `JS/OCR_OUTPUT/DOCX_OUTPUT` carried over from V2. `BARCODE` is a 0-byte no-op on the exe (linker `/OPT:REF` already dead-strips it) but kept off for build-time honesty. `BROTLI/HYPHEN/HYPHEN_ALL` are the only *referenced*-code feature levers that actually shrink the exe (−1.07 MB together). `OFFICE/MOBI/HTML/TXT/IMG` are non-allowlist document handlers (litepdf ships only `.pdf .epub .cbz .xps .fb2 .svg` — `Document.cpp:133`; EPUB/FB2 keep `FZ_ENABLE_HTML_ENGINE`, which is auto-enabled by the kept formats and is NOT pruned). `TOFU_SYMBOL/CJK_LANG` carried from V2. **`TOFU_NOTO` is the load-bearing lever** (−6.7 MB) that drops non-CJK Noto fallback — the locked decision A. `TOFU_EMOJI` is kept for spike-V9 parity but is a **no-op in 1.27.2** (spike V8 = 0 bytes; it has no standalone consult and is force-defined by `TOFU_NOTO` anyway) — it is therefore deliberately **excluded from the assertion** in Step 3. CJK fonts (Source Han + Droid Sans Fallback) and Base14 stay; M4 proved CJK renders tofu-free under this set.

- [ ] **Step 2: Drop the stale hard-coded `config.h` line ref from the block comment**

In the block comment above the prune (currently `:80-83`, "FZ_ENABLE_* are #ifndef-guarded (config.h:162-208)"), the line number is stale and will re-stale on every MuPDF bump. Replace the parenthetical with a forward pointer to the assertion, e.g.: "FZ_ENABLE_* are `#ifndef`/`#if !defined`-guarded in config.h and TOFU_* are `#ifndef`-consulted in the font source; exact lines vary per MuPDF version — the prune-effective assertion below verifies each guard/consult is still live." Keep the mechanism description.

- [ ] **Step 3: Add the UNCONDITIONAL, anchored prune-effective assertion**

Insert this block **immediately AFTER the closing `endif()` of the `if(NOT _cfg MATCHES "${_PRUNE_VER}")` block** (currently `:118`), so it runs on **every** configure — not only on the re-prune path (a marker-present warm/dirty reconfigure would otherwise skip it, exactly when post-bump macro drift matters). It re-reads the live files itself rather than depending on the in-block `_cfg_pristine`:

```cmake
# --- Prune-effective assertion (spec §5; Opus F6 + Codex #5) -----------------
# Runs on EVERY configure (outside the marker-guarded prune block) so a renamed/
# removed macro after a future MuPDF bump fails loudly HERE, not as a late,
# ambiguous size regression. Anchored to the actual guard/consult FORM (not a
# bare token) so a substring (FZ_ENABLE_HYPHEN ⊂ FZ_ENABLE_HYPHEN_ALL;
# FZ_ENABLE_HTML ⊂ FZ_ENABLE_HTML_ENGINE) or a comment-only opt-in line cannot
# mask a real removal. CMake regex has no \b, so we require a non-identifier
# char after the token.
file(READ "${_MUPDF_CONFIG_H}" _cfg_live)
foreach(_m FZ_ENABLE_JS FZ_ENABLE_OCR_OUTPUT FZ_ENABLE_DOCX_OUTPUT
           FZ_ENABLE_BARCODE FZ_ENABLE_BROTLI FZ_ENABLE_HYPHEN
           FZ_ENABLE_HYPHEN_ALL FZ_ENABLE_OFFICE FZ_ENABLE_MOBI
           FZ_ENABLE_HTML FZ_ENABLE_TXT FZ_ENABLE_IMG)
    # Accept #ifndef X  OR  #if !defined(X)  (BARCODE uses the latter at :331).
    if(NOT _cfg_live MATCHES "ifndef[ \t]+${_m}[^A-Za-z0-9_]"
       AND NOT _cfg_live MATCHES "defined[ \t]*\\([ \t]*${_m}[ \t]*\\)")
        message(FATAL_ERROR
            "Prune drift: no live #ifndef / #if-!defined guard for ${_m} in "
            "MuPDF config.h — it was renamed/removed upstream and the injected "
            "'#define ${_m} 0' is now a no-op. Re-audit the V9 set against this "
            "MuPDF version before proceeding.")
    endif()
endforeach()
# TOFU_* are consulted (bare #ifndef) in the FONT source, NOT config.h.
# font-table.h is the canonical consult site (font.c also references CJK_LANG).
# TOFU_EMOJI is intentionally NOT asserted: 1.27.2 has no standalone
# '#ifndef TOFU_EMOJI' consult — it is driven by TOFU_NOTO/NOTO_SMALL and
# measured 0 bytes in the spike (V8). It stays injected for spike-V9 parity.
file(READ "${_MUPDF_ROOT}/source/fitz/font-table.h" _ft_src)
foreach(_t TOFU_SYMBOL TOFU_NOTO TOFU_CJK_LANG)
    if(NOT _ft_src MATCHES "ifndef[ \t]+${_t}[^A-Za-z0-9_]")
        message(FATAL_ERROR
            "Prune drift: no live '#ifndef ${_t}' consult in "
            "source/fitz/font-table.h — the font-prune consult site moved/"
            "renamed upstream; the injected '#define ${_t}' is now a no-op.")
    endif()
endforeach()
message(STATUS "MuPDF prune-effective assertion passed (12 FZ_ENABLE_* + 3 TOFU_*; TOFU_EMOJI exempt).")
```
